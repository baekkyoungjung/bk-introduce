웹개발자 react native와 친구 되다

안녕하십니까? 자존감이 폭발하는 요즘 그냥 자랑 좀 해보고 싶어서 쓰는 글입니다. 물론 react native 설치법, 꿀팁 같은건 없고(react native 경력 2개월) 제가 느꼈던 react native 장단점과 보너스로 크몽에서 새롭게 선보인 단기 알바 매칭앱 SOON react native 개발기에 대해 겸손히 적어보려 합니다.

어떻게 react native로 개발 하게 되었는가

우선 별 볼일 없는 저를 소개하자면 개발 경력 3년 조금 넘고 React 2년 6개월, Vue 9개월 정도를 프론트 메인 라이브러리로 사용했습니다. 크몽에 입사한지는 10개월 쯤 되었네요.

개발자라면 react native (이하 RN)에 대해선 한번쯤 들어보셨을 겁니다. 저도 2년 전쯤 처음 들어봤는데요 그때는 네이티브앱에 비해 느리다, 성능을 못 따라간다, 역시 네이티브! 라는 말이 많아서 아 그런가 보다 하고 웹개발에만 집중했었죠. 그렇게 2018년 9월, 열심히 탕비실에서 Vue Component를 만들고 있던 중에 저희 크몽 CTO(a.k.a 크알)가 크몽에서 신규플랫폼 단기 알바앱을 기획중인데, 빠르게 시장반응을 확인하고, 개발 리소스를 최소화 하기위해 RN로 개발하면 어떨까하고 React를 경험했던 저에게 권유하셨습니다. 사실 그때 제 마음은 흥분의 도가니였습니다. 누군가에게 필요로 하는 사람이 된다는건 기분좋은 일이니까요.

그렇게 약 1주일간 RN을 필사적으로 공부하여 10월 초부터 본격적인 SOON 폭풍 개발을 시작했습니다. 기본적인 개발 스택은 python + RN + mobx 조합으로 구축되었습니다. (백엔드분들도 python으로 처음 도입!) 여러 레퍼런스들을 보며 나만의 best practice를 찾아갔고 mobx와의 조합도 훌륭했습니다. react는 익숙하지만 처음 앱 개발을 하는터라 수많은 시행착오를 겪어야했죠. 그만큼 새로운 경험도 엄청나게 했습니다. 불과 얼마전까지도 초창기에 (1달전..) 짯던 코드를 보고 한숨을 깊게 쉬고 리팩토링을 한게 수두룩 합니다. 그 사이 실력이 늘어났나보다! 라고 열심히 행복회로를 돌렸죠.

아무튼 우선 RN은 웹 개발자 (초급 이상의 javascript를 이용한다는 전제하에)라면 1분도 안걸려 hello world를 띄울수 있을 만큼 쉽게 만들어져 있습니다.

요즘은 expo라는 툴 덕분에 안그래도 쉽게 앱 개발을 쉽도록 만들어진 RN을 더더더욱 쉽게 접할 수 있도록 되어있습니다.

기본적으로 RN은 React기반으로 되어있기 때문에 나는 React를 못써~ 나는 vue or angular 밖에 안해봤어~ 라고 하더라도 충분히 빠르게 배울수 있을 것이라 생각합니다. React나 vue나 거기서 거기 (위험한 발언이지만 둘다 상용서비스로 사용해본 입장에선 하나 배우면 다른 라이브러리를 배우는 시간은 처음 배울때 대비하여 절반도 안걸렸던것 같네요)

앱 개발이라고 안해보기 보단 일단 hello world만 찍어보면 와 재밋다~ 하고 이것저것 더 해보는 자신의 모습을 볼 수 있을 겁니다. 앱개발을 위해서 RN을 해본다기 보다 React를 아주 재미있게 배울수 있는 도구로서도 훌륭합니다. 그냥 지루하게 docs 보면서 하는것 보단 전혀 새로운 분야를 배우면서 자연스레 React도 배울수 있습니다. Facebook에서 React를 내세우며 앱개발 RN도 할수 있다!의 기술력 과시가 아니라 RN은 정말 쓸만했습니다.

RN의 미친 장점

첫번째는 ios, android 동시 개발.

RN이 가지는 장점 딱 3개만 나열해 보겠습니다. 사실 하나의 코드로 ios, android가 만들어지기 때문이죠. 여기서 한술 더떠 view부분을 html, css로 변환후 몇몇 로직들만 수정하면 web으로도 그대로 가져올수 있습니다. 반대로 react로 만들어진 web기반 서비스를 react native로 변환도 가능합니다. RN이 접근한 Learn once, write anywhere이 뭔가 멋있었죠(물론 완벽하진 않지만..)

두번째는 미친 수준의 개발속도

React는 live-reload(코드가 변경되면 자동으로 새로고침)와 hot-reload(코드가 변경되면 변경된 딱 그 부분만 랜더링)를 지원합니다. 그 어떤 복잡한 설정 없이도요. 일단 RN은 compile, build 과정이라는게 없다고 봐도 되기 때문에(속도면에서) 구지 live, hot reload가 없이도 빠른 개발이 가능합니다. 하지만 저 두놈이 있기 때문에 코드를 수정하면 그 화면을 직접 보는데까지 오바좀 섞으면 1초도 채 안걸립니다(사실 1~5초 걸림). QA시에도 변경사항을 바로바로 확인하여 디자이너, 기획자와의 피드백을 바로바로 들을수 있어 UX/UI를 잡을 수 부분도 아주 효과적입니다. 상상보단 직접보는게 더 와닿으니까요. 거기다가 expo환경에서 개발하고 있다면 가상 simulator가 없어도, xcode, android studio를 건들지 않아도(SOON이 런칭되고 나서도 android studio는 아직 설치도 안했습니다.) 개발하고, 배포하는데 아무 지장이 없습니다. 이정도만 해도 장점이 꽤 큰데 사실 진짜 장점은 다음입니다.

마지막으로 OTA(실시간 배포)기능 입니다.

정말 이것이 제일 미친 장점입니다. RN으로 만들어진 앱은 기능이 추가되어도, 버그 수정 해도, 디자인이 바뀌거나 해도 심사를 거치지 않고 앱 실행시 언제나 최신 javascript를 다운로드 받고 실행하기 때문에 유저는 언제나 최신상태의 앱을 경험할 수 있습니다. 물론 몇가지 제한 사항이 있긴 합니다. (앱 아이콘이 바뀌거나 앱과 관련된 config가 바뀔시엔 심사 필요) 언제나 덤벙대고 맨날 까먹는 저는 정말 유용하게 쓰는 기능입니다. 항상 노트북을 가지고 다니기 때문에 뭔가 오류가 생기면 아 이부분 예외처리 깜빡했네? 수정하고 publish만 하면 되기때문에 오류에 대한 심리적인 부담감이 엄청나게 줄었습니다.

당연히 단점도 존재합니다.

RN은 성능이 아무래도 딸린다던데...

native 코드로 변환작업이 필수이기 때문에ㅜㅜ

태생이 네이티브가 아니기 때문에 생기는 해결하기 힘든(불가?) 단점이죠.. 저도 이 얘기를 수도 없이 들었습니다. 하이브리드앱, 웹앱등이 태생이 Swift와 Java등의 Native를 따라 갈수 있을리 만무했죠. RN이 세상에 나오고서도 하브, 웹앱보다는 빠르지만 네이티브와 비교하기엔 민망했다고 합니다. (사실 잘 모름) 그 이후에 주기적으로 성능 향상과 효율성에 대한 업데이트가 있었다는것? 정도.. 성능에 대해선 딱 이정도의 정보만 알고 있었고 SOON을 만들기 시작했습니다. 당연히 SOON에는 많은 기능이 담겨있진 않았고 오류 투성 이었지만 성능에 대해선 한번도 이슈가 된 적은 없었습니다. 물론 기능이 계속 추가되고 규모가 커지다보면 성능이 느려지겠지만 ms로 비교하여 테스트 하지 않는 이상 딱히 유의미한 결과라고 생각되진 않았습니다. 더군다나 SOON의 핵심가치인 빠르고 간편하게 단기알바를 매칭 시켜준다는 심플한 가치에서 시작되었기 때문에 이것저것 앱의 몸집이 아주 크게 늘어 날 것 같지 않다고 판단했기 때문에 SOON을 만들기엔 RN이 가장 최적이라 생각하셨던거 같습니다 (@CTO) 객관적으로 보면 아무리 RN이 나르고 긴다한들 네이티브에 성능적으로보면 대적 할 수 없을거라 생각합니다. 하지만 언어를 고르고 서비스를 생각한다기 보다 서비스 성격에 맞게 언어를 선택하는게 올바르다 생각합니다. 언어는 도구일 뿐이니까요 (참고자료 RN과 네이티브코드와의 성능 테스트) https://blog.wonhada.com/?p=2974

아무래도 javascript와 react에 대해 좀 친해야..

RN이 아무리 쉽게 앱개발을 할 수 있다지만 javascript와 React에 대해선 그래도 조금 알아야 러닝커브가 좀 낮다고 생각 되네요. 이 두가지를 잘 모르는 상태로 무턱대고 RN을 시작하면 막상 RN보다 javascript, React를 공부하다가 포기하는 경우가 많은것 같습니다.

사라지지 않는 네이티브에 대한 두려움

전 네이티브 코드와 환경을 전혀 모릅니다. 앱 등록시 인증서가 필요하다는것도 처음 알았고, 정말 거의 아무것도 몰랐습니다. 그래서 초기에 러닝커브가 꽤나 있었죠. Swift, Java를 공부한것은 아니지만 앱 등록, 배포는 어떻게 진행되는지, 하나의 앱이 존재하는 생태계 그런 전반적인 것들에 대해 많이 공부 했습니다. 개발 하다보면 native를 연결해서 해결하는 오류등을 맞이 하게 되는데, 그런 부분에선 전혀 알지 못하기 때문에 크몽 앱개발자의 도움이 많이 필요했습니다. 아직도 네이티브관련된 부분에서 에러터지면 겁부터 납니다. 제가 혼자 해결 할 수 없는 부분이 있기 때문에.. RN에 좀 더 적응이 되고 하면 조금씩 기초 앱 개발정도는 따라 할 수 있도록 공부 해야 할것 같습니다. 이러다 앱개발로 전향 할지도..
