개발자의 생활

배우는 즐거움

html, javascript, css 로 기초를 배웠고 bootstrap 으로 grid design과 생각하던 것 보다 훨씬 더 많은 것을 할 수 있다는 것을 알았다. 그리고 sass 와 emmet 으로 이들을 한층 더 편리하게 사용했다.

jquery 는 dom을 이해하기에 교과서와도 같았으며 react 는 view에 집중하고 복잡한 flow를 벗어나 front가 굴러가는 구조를 한눈에 볼 수 있었다. canvas, image, video, web audio context api, immutableJS, ESLint 등 필요한 건 가리지 않고 습득하며 점점 자신에 대한 경쟁력 그리고 자신감 팍팍 상승하였다. 그리고 요즘은 워낙 세상이 좋아져서 뭔가 필요하면 검색하여 찾고 (왠만한 건 다 있음), 없으면 만들었다.

개인 프로젝트를 하기 위해선 backend가 필요했기 때문에 nodejs를 공부하며 express를 사용하다가 좀 더 쉬운 meteor 를 이용하여 진행. 한동안 meteor로 업무, 개인 프로젝트를 진행하다가 laravel 로 이전. 전통적인 framework인 PHP를 공부하며 MVC에 대한 개념을 확실히 이해. 공부하다 보니 어떤 언어든 문법이 어쩌고저쩌고 보다 그 언어가 지향하는 그 어떤.. 그 사상이 훨씬 중요하다는 것을 깨닳음.

나아가고 싶은 방향

react와 더불어 좀 더 하드하게 사용할 frontend library (angular, vue 아직까진 토이용) react가 익숙하고 오래 사용해 왔기 때문에 아직은 무엇을 만들 때 react부터 머릿속에서 떠오르지만 frontend library의 선택지가 많아지면 용도에 맞게 수단을 선택할 수 있을 것 같다. 혹은 react를 더 강화하여 최적화, redux, mobx 등을 이용하여 좀 더 효율적인 상태 관리를 배워보고 싶다. (redux, mobx는 따로 공부했기 때문에 토이같은 소규모에만 적용해 봐서 아직은 서비스에 쓰긴 자신이 없다.)

backend와 database에 대해 조금 더 공부해보고 싶음. frontend에 필요한 data들은 결국 database에서부터 나오기 때문에 단순히 CRUD나 단일 table 설계가 아닌 미래를 예측/대비하는 전체를 보는 database 설계가 정말 중요하다고 느껴짐.

개발보다 더 중요한 것

최신 기술들의 개발 문서나 동영상은 대부분 영어로 되어 있기 때문에. 이거 안 되겠다 싶어 작년 11월쯤부터 영어 공부에 빠져 영어 그룹 스터디도 하며 공부 중이지만 역시나 영알못에게 쉽지 않은 도전. 꾸준히 장기전으로 봐야 할 것 같다. react 공부할 시기엔 react 모임에 참석하여 약 5개월을 진행하였고 동시에 업무에서 2년가량 react로 개발, javascript 패턴과 테스트 책 기반으로 TDD에 대해 4개월간 스터디 참여했다.

카페 같은 곳은 돈이 남아도는 된장남이나 가는 곳이고 맥북을 꺼내 허세에 찌들어 앉아 있는 곳이라고 생각했던 때가 있었는데 지금은 커피와 함께하는 카페 공부가 세상 평화롭고 좋다. 회사에서도 인정받는 만큼 보답하기 위해 더 노력했던 것 같다. 그러면서 점점 나의 생활 패턴도 바뀌어 갔다.

개발자가 되기 전 게을렀던 내가 지금은 시간을 쪼개서 출근 전 집이든 커피숍이든 사무실이든 업무나 공부, 가계부 정리 등 필요한 것을 한 뒤 출근을 한다. 그렇게 하지 않으면 왠지 뒤처지는 느낌도 나고, 경쟁력을 잃고 싶지 않다. 사실 개발자로 전향하고 나서 개발보다 현상에 대해 사고하고 고민하고 상상하는 방법을 훨씬 더 많은 것을 배웠다. 그리고 그런 습관은 개발할 때도, 사회생활 속에서도 많은 도움이 되었고 나의 인생에 아주 큰 부분이 되었다.

사회 생활

회사를 위해 일하지 말자

너무 어려워서 포기했던 library 를 6개월 뒤 다시 도전했을 때 쉽게 느껴진 경험, 그동안 몰라서 그냥 지나쳤던 것들이 개발 용어들과 개념들을 알아가며 점점 보이기 시작했을 때, 생활코딩과 기타 강의 동영상들을 정주행해가며 하나하나 따라 치고 동영상 아재의 화면이 나의 화면에도 떳을 때, 정말 아드레날린이 분출하는걸 느낄 수 있었다.

그동안 회사를 위해 일한 것이 아닌 나를 위해 일하다 보니 점점 회사도 같이 성장해 나갔다. 만약 회사에 필요한 일만 하고 새로 나온 기술들을 회사에 필요 없다고 신경 쓰지 않았더라면 이런 경험을 하지도 못했을 것이다. 개인 프로젝트를 진행하며 더 나은 방법이나 미립자 팁 일지라도 팀원에게 공유하였을 때 점점 나와 팀이 발전하는 것을 느꼈다.

팀원과 팀웍

사실 이력서를 쓰면서 어떤 식으로 나를 어필 해야 할지 많은 고민을 했다. 특히 능력을 적는 항목에서 그랬는데, 문제 해결 능력 탁월, 소통, 가파른 러닝 커브, 창의/비판적 사고 등등 수많은 능력치를 적었다. 하지만 이 페이지에서 주절주절 이런 글들을 적다 보니 그게 무슨 소용이 있나 싶다. 혼자 토니 스타크처럼 대단하더라도 소통이 안 된다면 서로 간의 믿음이 무너지고 만다.

사람들이 모인 곳은 언제나 크고 작은 트러블들이 있다. 내가 정말 인생의 한번 일지라도 큰 잘못을 저지르는 일을 하지 않은 이상 언제나 답은 있었다. 나의 경우는 내가 느끼는 감정이 어떻든 상대방의 입장에 완전히 빙의해보려고 노력해본다. 대부분 경우엔 당사자와 뭐 술 한잔하거나 서로 감정 소모 없이 솔직하게 말하며 좋은 방향으로 비교적 쉽게 해결해 나가거나 그게 힘들다면 이해해줄 수 있는 제 3자에게 털어놓아 어떻게 하는 게 좋을지 조언을 구했던 것 같다. 절대 혼자 해결할 수 없는 일은 혼자 끙끙 앓진 않았다. 개인적인 감정을 업무에서 풀거나 겉으로 표현해서는 안 된다고 생각한다. 물론 무감각 인간이 아닌 이상 항상 잘 되진 않지만, 항상 해결해 왔기 때문에 큰 걱정은 하지 않는다. 지금은 조금 더 나 자신을 좀 가꾸고, 성장해나가다 보면 좋은 회사가 아닌 좋은 사람들을 만날 수 있을 것이라 믿어 의심치 않는다.
